## 3. 별표(⭐) 핵심 페이지

다음 4개가 ClassAuto의 결정적 페이지입니다:

| 페이지 | 결정적 이유 |
|---|---|
| `/demo` | 베타 보급 전환 핵심 |
| `/v/[강의ID]` | 학생이 처음 접하는 화면 |
| `/studio` | 교수자가 가장 자주 쓰는 워크플로우 |
| `/dashboard` | 교수자 첫 화면 |

이 4개의 사용자 경험이 무너지면 다른 페이지가 아무리 좋아도 제품이 작동하지 않습니다.

## 4. 작업 순서 권장 (페르소나 흐름 기반)

| 단계 | 작업 묶음 | 이유 |
|:---:|---|---|
| **1단계** | demo + 동적 요소 개선(기존 4페이지) | 이미 결정 완료, 가장 가시적 효과 |
| **2단계** | 학생 진입 3종 (`/v/[ID]` · 가입 · 온보딩) + 영상 시청 통합 | 학생 흐름 한 번에 완성 |
| **3단계** | 교수자 온보딩 + dashboard + studio 보강 | 교수자 첫 사용 흐름 완성 |
| **4단계** | trust + security + use-cases + beta-apply + contact | 영업·신뢰 페이지 일괄 |
| **5단계** | legal/terms · privacy · help · changelog | 출시 직전 마무리 |
| **6단계** | account/billing · team · settings · profile · integrations · status | 유료화 후 |
| **7단계** | 영문 + partners + community | Phase 5 |

페르소나(잠재 가입자 → 학생 → 교수자 → 학과장)의 흐름이 끊기지 않도록 묶음 단위로 작업합니다.

## 5. 페이지 간 전환 흐름

### 5.1 진입점별 매트릭스

| 진입점 | 가능한 흐름 | 핵심 페이지 |
|---|---|---|
| 동료 추천 (카톡 링크) | demo → CTA → 베타 신청 폼 | `/demo`, `/beta-apply` |
| 검색 유입 | 랜딩 → features → pricing → 베타/가입 | `/`, `/features`, `/pricing` |
| 학과장 도입 | 랜딩 → use-cases → 기관 라이선스 → 견적 폼 | `/`, `/use-cases`, `/contact` |
| 학생 진입 | 강의 링크 → 학생 가입 → 영상 시청 | `/v/[ID]`, 영상 시청 |

각 흐름이 끊기지 않도록 페이지 간 연결을 설계해야 합니다.

### 5.2 시각적 톤 전환

| 페이지 그룹 | 톤 |
|---|---|
| 공개 (랜딩, features, pricing 등) | 다크 베이스 + 골드 |
| 데모 페이지 | 다크 모드 강제 (학습자 시점 명시) |
| 교수자 영역 | 라이트 베이스 + 골드 + 데이터 시각화 |
| 학습자 영역 | 다크 모드 강제 (`#0A0A0A`) |

라이트 ↔ 다크 전환 시 페이지 자체에서 부드럽게 처리.
