## 1. 문제 정의 — 단가가 매출을 넘는다

요금 정책 문서(`01-pricing-policy.md:13`)의 자기 숫자로 검증한 현재 단위 경제:

| 항목 | 값 | 출처 |
|---|---|---|
| 영상 1편 **원가** | 약 ₩11,000 | `01-pricing-policy.md:13` |
| Basic 플랜 편당 **매출** | ₩2,375 (₩19,000 / 8편) | `01-pricing-policy.md:14` |
| **편당 손익** | **−₩8,625 (약 4.6배 적자)** | 계산 |

테스트에서 ₩15,000이 나온 것은 사고가 아니라 이 구조의 정상 동작 결과다. 흑자 전환의 조건은 **원가 ₩11,000 → 약 ₩1,500 이하(약 −85%)**.

> **환율 가정**: 본 문서의 ₩ 환산은 $1 = ₩1,380, `HEYGEN_COST_USD_PER_SECOND = 0.0083`(`core/config.py:106`) 기준. 즉 **HeyGen ≈ ₩11.45/초 ≈ ₩687/분**. 실제 결제 청구액으로 추후 보정한다.

## 2. 비용 구조 진단 (근본 원인 3가지)

### 2.1 HeyGen 아바타를 슬라이드마다 렌더한다 (최대 원인)

- `video.py:375`(승인 시), `render.py:120`(직접 렌더 요청) — 강의 1편을 **세그먼트(슬라이드) 개수만큼** `VideoRender` 생성 후 각각 `render_slide` → HeyGen `create_video` 호출.
- 30슬라이드 강의 = HeyGen 아바타 영상 30개.
- 가드레일 문서(`02-guardrails.md:30`)는 "HeyGen 영상 합성 = ❌ 1회"로 **가정**하나 실제 구현은 N회. 기획 가정과 코드가 어긋나 있다.
- 부작용:
  - HeyGen은 영상 1건마다 **최소 과금/크레딧 올림**이 있어 짧은 클립 N개가 긴 1개보다 비싸다.
  - N슬라이드 중 일부가 실패하면 앞서 성공한 슬라이드는 **이미 청구된 채로 버려진다** → "결과물 미확보, 비용만 발생"의 직접 원인.

### 2.2 해상도 1080p 하드코딩

- `heygen.py:143` → `"dimension": {"width": 1920, "height": 1080}`.
- HeyGen은 대부분 요금제에서 1080p를 720p의 **약 2배 크레딧**으로 청구. 강의 슬라이드는 720p로 충분.

### 2.3 기본 TTS가 ElevenLabs (Google 대비 ~19배)

- 단가(`cost_tracker.py:28-29`): ElevenLabs $0.30/1K자 vs Google TTS $0.016/1K자.
- `render.py:140`은 TTS 비용을 `cost_usd=0.0`으로 기록 → **회계가 TTS 비용을 누락**. 실제 원가는 ₩11,000보다 높을 수 있다.

## 4. 최종 영상 모델 (확정 2026-05-27)

교수자 아바타 강의는 불쾌한 골짜기 완화·신뢰성 측면에서 **유지하되**, 아바타를 영상 전체가 아니라 핵심 구간에만 투입한다.

```
[인트로 아바타 30~60s]   ← HeyGen (강의당 1회)
[슬라이드1 + TTS 음성]   ┐
[슬라이드2 + TTS 음성]   │  ← ffmpeg 합성, HeyGen 비용 0
   ...                   ┘
[아웃트로 아바타 30~60s]  ← HeyGen (강의당 1회)
```

- 강의 본문 HeyGen 렌더: **N → 2 고정**.
- 본문은 `step1`이 이미 S3에 올리는 슬라이드 PNG(`pipeline.py:152` `_render_and_upload_slide_images`) + TTS 음성을 ffmpeg로 합성.
- 차별점 강화: "원가 투명·저비용"을 마케팅 메시지로 활용 가능(`CLAUDE.md` §정체성 4가지 중 "비용 투명성").

### 4.1 본문 컴포지터 비용 (목표 검증)

10분 강의, 한국어 발화 ~5자/초(`pipeline.py:279`), 음성 1.3배속 가정:

| 구성 | 계산 | 비용 |
|---|---|---|
| 인트로+아웃트로 아바타 90s @720p | 90s × ₩11.45 × 0.5(720p) | 약 ₩515 |
| 본문 TTS (Google, ~3,000자) | 3,000 × $0.016/1K × ₩1,380 | 약 ₩66 |
| Claude 스크립트 생성 | 슬라이드 N장, max_tokens 2048 | 약 ₩200 |
| S3 저장/전송 | — | 약 ₩100 |
| **합계** | | **약 ₩900** |

→ Basic 편당 매출 ₩2,375 대비 흑자.

## 5. 아바타 Q&A 모델 (학생 질문 → 중앙 아바타 답변)

### 5.1 동작

```
학생 질문 → RAG 텍스트 답변(무료, 즉시 표시)
          → [아바타로 듣기] 탭 시에만
          → 강의 일시정지 → 화면 중앙 아바타 답변 재생
          → 캐시 적중 시 0원 / 미스 시 HeyGen 1렌더 (플랜별 상한 차감)
```

플랜별 아바타 직접 답변 상한(제안): **Free 1 / Basic 2 / Pro 3** — 단위는 **학생 1명 × 영상 1편**(기존 채팅 Q&A 한도 `02-guardrails.md:130`과 일관).

### 5.2 ⚠️ 비용 함정 — 학생 인터랙션 경로의 HeyGen

가드레일 문서의 경고(`02-guardrails.md:40`): "영상 생성 비용보다 **학생 인터랙션 비용이 더 빠르게 누적**된다." 아바타 Q&A는 HeyGen을 다시 이 경로에 넣는 것이라 가장 위험한 구역이다.

**영상당 Q&A 아바타 비용 시뮬레이션** (Basic 강의 80명, 답변 ~20초 가정):

| 시나리오 | 계산 | 영상당 비용 |
|---|---|---|
| 캐싱·옵트인 없음 (1080p) | 80×2 × 20s × ₩11.45 | 약 ₩36,600 |
| + 720p | ×0.5 | 약 ₩18,300 |
| + 텍스트 우선(아바타 옵트인 30%) | ×0.3 | 약 ₩5,500 |
| + 클러스터 캐시(적중 80%) | ×0.2 | **약 ₩1,100** |

→ 상한(1/2/3회)만으로는 못 막는다. 상한은 학생 1명을 막을 뿐, 학생 수가 곱해진다. **캐싱이 필수.**

### 5.3 비용 통제 장치 (3중)

1. **클러스터 답변 캐시** (가장 중요) — 아바타 답변 렌더 전, 이 영상에서 이미 생성한 답변들과 질문 임베딩 코사인 유사도 비교. 임계값(제안 0.9) 이상이면 **새 렌더 없이 기존 클립 재생**(HeyGen 0). RAG가 이미 질문을 임베딩하므로(`embedding.py`) 한 겹만 추가. Pro 전용 "유사 질문 클러스터링"(`01-pricing-policy.md:128`)을 아바타 레이어로 확장.
2. **텍스트 우선, 아바타는 탭으로** — RAG 텍스트 답변은 항상 무료 즉시 표시. 학생이 "아바타로 듣기"를 누를 때만 크레딧 차감. 대부분 누르지 않음.
3. **강의/계정 단위 예산 캡** — Phase 0의 서킷 브레이커가 절대 상한(백스톱).

### 5.4 미해결 결정사항

- [ ] **상한 단위**: "학생×영상" 확정? (월 단위/일 단위 병행 여부)
- [ ] **캐시 재생도 크레딧 차감?** 차감(단순·보수) vs 무료(관대·UX 우선)
- [ ] **캐시 유사도 임계값**: 0.9 적정? 베타에서 오답 재생 빈도로 보정
- [ ] **답변 길이 상한**: 아바타 답변 max 초(예: 30초)로 클램프해 변동비 상한 고정
