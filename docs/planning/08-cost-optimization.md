## 1. 문제 정의 — 단가가 매출을 넘는다

요금 정책 문서(`01-pricing-policy.md:13`)의 자기 숫자로 검증한 현재 단위 경제:

| 항목 | 값 | 출처 |
|---|---|---|
| 영상 1편 **원가** | 약 ₩11,000 | `01-pricing-policy.md:13` |
| Basic 플랜 편당 **매출** | ₩2,375 (₩19,000 / 8편) | `01-pricing-policy.md:14` |
| **편당 손익** | **−₩8,625 (약 4.6배 적자)** | 계산 |

테스트에서 ₩15,000이 나온 것은 사고가 아니라 이 구조의 정상 동작 결과다. 흑자 전환의 조건은 **원가 ₩11,000 → 약 ₩1,500 이하(약 −85%)**.

> **⚠️ 단가 정정 (2026-05-27 조사)**: `HEYGEN_COST_USD_PER_SECOND = 0.0083`(= $0.50/분, `core/config.py:106`)은 **실측의 절반**이다. HeyGen API 공식 단가는 **표준 $1/분(720p/1080p)**, 프리미엄 Avatar IV는 **$4/분**. 즉 실제 원가는 ₩11,000보다 높고, ₩15,000 사고는 프리미엄 아바타 사용 시 충분히 설명된다. **본 문서 ₩ 환산은 $1 = ₩1,380, HeyGen 표준 $1/분 = ₩1,380/분 ≈ ₩23/초 기준**으로 통일한다(config 값도 0.0167로 정정 필요 — Phase 0).

## 2. 비용 구조 진단 (근본 원인 3가지)

### 2.1 HeyGen 아바타를 슬라이드마다 렌더한다 (최대 원인)

- `video.py:375`(승인 시), `render.py:120`(직접 렌더 요청) — 강의 1편을 **세그먼트(슬라이드) 개수만큼** `VideoRender` 생성 후 각각 `render_slide` → HeyGen `create_video` 호출.
- 30슬라이드 강의 = HeyGen 아바타 영상 30개.
- 가드레일 문서(`02-guardrails.md:30`)는 "HeyGen 영상 합성 = ❌ 1회"로 **가정**하나 실제 구현은 N회. 기획 가정과 코드가 어긋나 있다.
- 부작용:
  - HeyGen은 영상 1건마다 **최소 과금/크레딧 올림**이 있어 짧은 클립 N개가 긴 1개보다 비싸다.
  - N슬라이드 중 일부가 실패하면 앞서 성공한 슬라이드는 **이미 청구된 채로 버려진다** → "결과물 미확보, 비용만 발생"의 직접 원인.

### 2.2 해상도 1080p 하드코딩

- `heygen.py:143` → `"dimension": {"width": 1920, "height": 1080}`.
- HeyGen은 대부분 요금제에서 1080p를 720p의 **약 2배 크레딧**으로 청구. 강의 슬라이드는 720p로 충분.

### 2.3 기본 TTS가 ElevenLabs (Google 대비 ~19배)

- 단가(`cost_tracker.py:28-29`): ElevenLabs $0.30/1K자 vs Google TTS $0.016/1K자.
- `render.py:140`은 TTS 비용을 `cost_usd=0.0`으로 기록 → **회계가 TTS 비용을 누락**. 실제 원가는 ₩11,000보다 높을 수 있다.

## 4. 최종 영상 모델 (확정 2026-05-27)

교수자 아바타 강의는 불쾌한 골짜기 완화·신뢰성 측면에서 **유지하되**, 아바타를 영상 전체가 아니라 핵심 구간에만 투입한다.

```
[인트로 아바타 30~60s]   ← 립싱크 렌더 (강의당 1회)
[슬라이드1 + TTS 음성]   ┐
[슬라이드2 + TTS 음성]   │  ← ffmpeg 합성, 아바타 비용 0
   ...                   ┘
[아웃트로 아바타 30~60s]  ← 립싱크 렌더 (강의당 1회)
```

- 강의 본문 아바타 렌더: **N → 2 고정**.
- 본문은 `step1`이 이미 S3에 올리는 슬라이드 PNG(`pipeline.py:152` `_render_and_upload_slide_images`) + TTS 음성을 ffmpeg로 합성.
- 차별점 강화: "원가 투명·저비용"을 마케팅 메시지로 활용 가능(`CLAUDE.md` §정체성 4가지 중 "비용 투명성").

### 4.1 본문 비용 (목표 검증, HeyGen 표준 $1/분 720p 기준)

10분 강의, 한국어 발화 ~5자/초(`pipeline.py:279`), 음성 1.3배속 가정:

| 구성 | 계산 | 비용 |
|---|---|---|
| 인트로+아웃트로 아바타 90s @720p | 90s × ₩23/s | 약 ₩2,070 |
| 본문 TTS (Google, ~3,000자) | 3,000 × $0.016/1K × ₩1,380 | 약 ₩66 |
| Claude 스크립트 생성 | 슬라이드 N장, max_tokens 2048 | 약 ₩200 |
| S3 저장/전송 | — | 약 ₩100 |
| **합계 (HeyGen)** | | **약 ₩2,400** |
| **합계 (self-host 립싱크, 90s ≈ ₩30)** | | **약 ₩400** |

→ HeyGen으로도 슬라이드별 대비 −80%. self-host 전환 시 Basic 매출 ₩2,375 대비 큰 흑자(§6).

## 5. 아바타 Q&A 모델 (학생 질문 → 중앙 아바타 답변)

### 5.1 동작

```
학생 질문 → RAG 텍스트 답변(무료, 즉시 표시)
          → [아바타로 듣기] 탭 시에만
          → 강의 일시정지 → 화면 중앙 아바타 답변 재생
          → 캐시 적중 시 0원 / 미스 시 립싱크 1렌더
```

### 5.2 ⚠️ 비용 함정 — 학생 인터랙션 경로의 아바타

가드레일 문서의 경고(`02-guardrails.md:40`): "영상 생성 비용보다 **학생 인터랙션 비용이 더 빠르게 누적**된다." 아바타 Q&A는 렌더를 이 경로에 넣는 것이라 가장 위험한 구역이다. 상한(영상당 N개)만으로는 학생 수가 곱해지는 것을 막지 못한다 → **캐싱이 필수.**

### 5.3 비용 통제 장치 (채택)

1. **클러스터 답변 캐시** (가장 중요) — 아바타 답변 렌더 전, 이 영상에서 이미 생성한 답변들과 질문 임베딩 코사인 유사도 비교. 임계값(0.9) 이상이면 **새 렌더 없이 기존 클립 재생**(렌더 0). RAG가 이미 질문을 임베딩하므로(`embedding.py`) 한 겹만 추가. Pro 전용 "유사 질문 클러스터링"(`01-pricing-policy.md:128`)을 아바타 레이어로 확장.
2. **텍스트 우선, 아바타는 탭으로** — RAG 텍스트 답변은 항상 무료 즉시 표시. 학생이 "아바타로 듣기"를 누를 때만 렌더 후보. 대부분 누르지 않음.
3. **이중 상한 + 예산 캡** — 아래 베타 운영 모델의 영상·편수 상한 + Phase 0 서킷 브레이커(백스톱).

### 5.4 베타 운영 모델 (확정 2026-05-27)

베타는 **A/B 비교 실험**으로 운영한다. 교수자 1명 / 월 기준:

| 항목 | 값 |
|---|---|
| 월 강의 영상 생성 | 총 **6편** |
| ┝ 채팅 전용 (A군) | **4편** — 학습자는 텍스트 채팅 Q&A만 |
| ┕ 아바타 Q&A (B군) | **2편** — §5.1 구조 활성 |
| 아바타 Q&A 영상의 질문 상한 | **영상당 3개** |
| 과금 | **월 정액, 크레딧 차감 없음** |
| 실험 종료·의견 수렴 | **2026년 9월 학기 종료 후** |

목적: 아바타 답변(B)과 채팅 답변(A)의 학습 효과·만족도·집중도를 같은 교수자 안에서 비교. 정식 가격·상한은 이 데이터로 확정.

### 5.5 결정 확정 사항 (이전 미해결 항목 해소)

- [x] **상한 단위** → **이중 상한**: 월 2편만 활성 + 활성 영상당 3개. (학생×영상 단위 카운트는 캐시 적중 포함 여부와 함께 구현 시 확정)
- [x] **크레딧 차감** → **없음** (월 정액제 본질 유지, `01-pricing-policy.md:18` 단순성 원칙과 일치)
- [x] **캐시 유사도 임계값** → 0.9로 시작, 베타에서 오답 재생 빈도로 보정
- [ ] **답변 길이 상한** → 30초 클램프 권장(변동비 상한 고정). 확정 대기
- [ ] **캐시 적중도 상한(3개)에 카운트할지** → 차감 안 하므로 "교수자 비용 노출"은 없음. UX상 학생에게 보일 카운터 정의만 필요

## 6. 립싱크 아바타 제공자 전략 (2026-05-27 조사)

핵심 질문: "불쾌한 골짜기 완화가 목적이라면 꼭 비싼 HeyGen이어야 하나?"

### 6.1 상용 제공자 비교 (720p 표준, $1=₩1,380)

| 제공자 | 분당 비용 | 방식 | 한국어 | API | 비고 |
|---|---|---|:---:|:---:|---|
| **HeyGen 표준** | $1 (₩1,380) | 풀 아바타 | ○ | ○ | turnkey 최고품질 |
| HeyGen Avatar IV | $4 (₩5,520) | 프리미엄 | ○ | ○ | 비쌈 — **회피** |
| D-ID | ~$3 (₩4,140) | 사진→토킹헤드 | ○ | ○ | HeyGen보다 비쌈 |
| sync.so (lipsync-2) | $2.4~3 | 립싱크 전용 | ○ | ○ | 베이스 영상 필요 |
| DeepBrain AI | 엔터프라이즈 | 풀 아바타 | ◎(한국) | ○ | 한국어 강점, 비용 이점 X |
| **self-host OSS**<br>(MuseTalk / LatentSync 1.5) | **~$0.01~0.02**<br>(₩14~28) | 립싱크 | ○ | 자체 | **50~100배 저렴**, GPU 시간만 |

### 6.2 결론

1. **HeyGen 표준 $1/분은 상용 turnkey 중 오히려 싼 축.** D-ID·sync.so·VEED로 교체해도 비용 절감 없음(동급~더 비쌈). **제공자 교체는 답이 아니다.**
2. **진짜 비용 바닥 = self-host 오픈소스 립싱크.** ByteDance **LatentSync 1.5**(품질 최상, 2025-03), Tencent **MuseTalk**(실시간 30fps+, V100)에 교수자 실제 영상/사진 + TTS 음성을 넣어 립싱크. 마진은 GPU 시간(~₩20/분)뿐.
3. **불쾌한 골짜기**: 합성 스톡 아바타보다 **"교수자 실제 얼굴 베이스 + 립싱크"가 더 자연스럽고 더 싸다.** 입만 움직이는 진짜 얼굴이므로. 이미 `upload_talking_photo`(`heygen.py:220`) 인프라 존재.

### 6.3 단계적 전략

- **베타(지금)**: HeyGen 유지. 인트로/아웃트로만 쓰니 저볼륨 → 절대 비용 작고 첫인상 품질은 turnkey가 안전. Avatar IV(프리미엄) 금지, 표준 720p 사용. config 단가 실측($1/분=0.0167) 정정.
- **추상화(Phase 0~1)**: `LipSyncProvider` 인터페이스 추출. `heygen.py`가 이미 격리돼 있어 교체가 *config 한 줄*이 되도록.
- **정식/스케일(Phase 3)**: Q&A 답변 → 전체 클립 순으로 self-host(MuseTalk/LatentSync)로 이전. 교수자 본인 영상 1개 등록 → 모든 인트로/아웃트로·Q&A를 그 얼굴로 립싱크.
