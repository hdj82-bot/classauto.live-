## 1. 문제 정의 — 단가가 매출을 넘는다

요금 정책 문서(`01-pricing-policy.md:13`)의 자기 숫자로 검증한 현재 단위 경제:

| 항목 | 값 | 출처 |
|---|---|---|
| 영상 1편 **원가** | 약 ₩11,000 | `01-pricing-policy.md:13` |
| Basic 플랜 편당 **매출** | ₩2,375 (₩19,000 / 8편) | `01-pricing-policy.md:14` |
| **편당 손익** | **−₩8,625 (약 4.6배 적자)** | 계산 |

테스트에서 ₩15,000이 나온 것은 사고가 아니라 이 구조의 정상 동작 결과다. 흑자 전환의 조건은 **원가 ₩11,000 → 약 ₩2,400 이하(약 −80%)**.

> **⚠️ 단가 정정 (2026-05-27 조사)**: `HEYGEN_COST_USD_PER_SECOND = 0.0083`(= $0.50/분, `core/config.py:106`)은 **실측의 절반**이다. HeyGen API 공식 단가는 **표준 $1/분(720p/1080p)**, 프리미엄 Avatar IV는 **$4/분**. 실제 원가는 ₩11,000보다 높고, ₩15,000 사고는 프리미엄 아바타 사용 시 충분히 설명된다. **본 문서 ₩ 환산은 $1 = ₩1,380, HeyGen 표준 $1/분 = ₩1,380/분 ≈ ₩23/초 기준**(config 값도 0.0167로 정정 — Phase 0).

## 2. 비용 구조 진단 (근본 원인 3가지)

### 2.1 HeyGen 아바타를 슬라이드마다 렌더한다 (최대 원인)

- `video.py:375`(승인 시), `render.py:120`(직접 렌더 요청) — 강의 1편을 **세그먼트(슬라이드) 개수만큼** `VideoRender` 생성 후 각각 `render_slide` → HeyGen `create_video` 호출.
- 30슬라이드 강의 = HeyGen 아바타 영상 30개.
- 가드레일 문서(`02-guardrails.md:30`)는 "HeyGen 영상 합성 = ❌ 1회"로 **가정**하나 실제 구현은 N회. 기획 가정과 코드가 어긋나 있다.
- 부작용:
  - HeyGen은 영상 1건마다 **최소 과금/크레딧 올림**이 있어 짧은 클립 N개가 긴 1개보다 비싸다.
  - N슬라이드 중 일부가 실패하면 앞서 성공한 슬라이드는 **이미 청구된 채로 버려진다** → "결과물 미확보, 비용만 발생"의 직접 원인.

### 2.2 해상도 1080p 하드코딩

- `heygen.py:143` → `"dimension": {"width": 1920, "height": 1080}`.
- HeyGen은 대부분 요금제에서 1080p를 720p의 **약 2배 크레딧**으로 청구. 강의 슬라이드는 720p로 충분.

### 2.3 TTS — 비용보다 "발음 품질"이 우선 (전공 특수성)

- 단가(`cost_tracker.py:28-29`): ElevenLabs $0.30/1K자 vs Google TTS $0.016/1K자.
- 그러나 **본 서비스 1차 사용자는 중어중문 교수자** — 스크립트에 한자·중국어 문장이 혼용된다(`script_generator.py`: "중국어는 한자 그대로"). **한국어 TTS가 한자를 읽으면 한국식 한자음으로 오발음**해 강의 내용이 틀어진다.
- **결정(2026-05-27): 기본 TTS = ElevenLabs v3.** v2는 중국어 발음이 뭉개져 양보 불가. 비용 절감 레버를 TTS에서 빼고 **아바타(HeyGen→self-host) + mp4 굽기 제거**에 집중한다.
- ⚠️ **ElevenLabs v3 공개 API는 아직 "coming soon"(early access)** — 빌드 전 프로덕션 API 접근 가능 여부 확인 필수. 미가용 시 fallback(언어 태그 v2 등) 백업. (출처: elevenlabs.io/blog/eleven-v3)
- `render.py:140`이 TTS 비용을 `cost_usd=0.0`으로 기록 → 회계 누락. `cost_tracker`로 실제 기록 필요(Phase 0).

## 4. 최종 영상 모델 (확정 2026-05-27)

교수자 아바타는 **실제 얼굴 1회 녹화 → 전 구간 립싱크** 모델로 통합. 영상 전체가 아니라 인트로/아웃트로 + Q&A에만 투입. **본문은 영상으로 굽지 않고 클라이언트에서 슬라이드쇼로 재생.**

```
[인트로 아바타 30~60s]   ← 립싱크 클립 (강의당 1회)
[슬라이드1 + TTS 음성]   ┐
[슬라이드2 + TTS 음성]   │  ← 클라이언트 슬라이드쇼, 영상 인코딩 0
   ...                   ┘
[아웃트로 아바타 30~60s]  ← 립싱크 클립 (강의당 1회)
```

### 4.1 클라이언트 슬라이드쇼 (mp4 굽기 제거)

본문은 ffmpeg로 합성하지 않고, 플레이어가 **슬라이드 이미지 + 구간별 TTS 음성 + 타임라인 JSON**(`VideoScript.segments`)을 동기 재생한다. 진짜 영상 파일은 인트로/아웃트로 아바타 클립뿐.

| 항목 | mp4 굽기 | 슬라이드쇼 |
|---|---|---|
| 서버 인코딩(ffmpeg) | 필요(비용·시간·실패) | **불필요** |
| 스크립트 수정 | 전체 재렌더 | **바뀐 구간만 재-TTS** |
| 슬라이드별 분석 | 추출 어려움 | **재생이 구간 단위 → 네이티브** |
| 인터스티셜 퀴즈/Q&A 일시정지 | 별도 오버레이 | 플레이어 상태 전환 |
| 자막·번역 | 영상에 burn | 텍스트 레이어 교체 |

- SCORM/HTML5 이러닝, Reveal.js + 내레이션과 동일한 검증된 구조.
- `step1`(`pipeline.py:152`)이 이미 슬라이드 PNG를 S3에 업로드 → 재사용.
- mp4 다운로드 약속(`01-pricing-policy.md:217`)은 **요청 시 on-demand ffmpeg 합성**으로 처리.
- **재생 구간 히트맵·슬라이드별 체류**가 공짜로 따라옴 → 논문/대시보드 자산([09-beta-program.md](./09-beta-program.md) §3).

### 4.2 본문 비용 (목표 검증, 10분 강의, ElevenLabs v3, 720p)

| 구성 | 계산 | 비용 |
|---|---|---|
| 인트로+아웃트로 아바타 90s @720p (HeyGen $1/분) | 90s × ₩23/s | 약 ₩2,070 |
| 본문 TTS (ElevenLabs ~3,500자, 강의당 1회) | 3,500 × $0.30/1K × ₩1,380 | 약 ₩1,449 |
| Claude 스크립트 생성 | 슬라이드 N장 | 약 ₩200 |
| 본문 영상 인코딩 | (슬라이드쇼 — 없음) | ₩0 |
| **합계 (HeyGen + ElevenLabs)** | | **약 ₩3,700** |
| **합계 (self-host 립싱크 시, 90s ≈ ₩30)** | | **약 ₩1,700** |

→ 슬라이드별 렌더(₩11,000+) 대비 −70%. self-host 전환 시 Basic 매출 ₩2,375 대비 흑자. TTS는 강의당 1회성이며 수정 시 변경 구간만 재합성.

## 5. 아바타 Q&A 모델 (학생 질문 → 중앙 아바타 답변)

상세 운영 모델(월 2영상 × 영상당 3렌더 배치, A/B)은 [09-beta-program.md](./09-beta-program.md) §5 참조. 본 절은 비용 통제 관점만.

### 5.1 동작 + 지연 대응

```
학생 질문 → RAG 텍스트 답변(즉시·무료)
          → 캐시 미스: 텍스트만 + 질문을 클러스터링 큐에 적립
          → 야간 배치: 상위 3개 클러스터만 아바타 렌더 (선착순 아님)
          → 다음날 유사 질문: 캐시에서 아바타 즉시 제공
```

HeyGen 렌더는 수 분 소요 → 실시간 렌더는 UX상 불가(학습자 이탈). **캐시 적중/사전 배치 렌더만 즉시 제공.** 비용은 교수자당 월 6렌더(2영상 × 3)로 고정.

### 5.2 ⚠️ 비용 함정 — 학생 인터랙션 경로의 아바타

가드레일 문서 경고(`02-guardrails.md:40`): "학생 인터랙션 비용이 영상 생성 비용보다 빠르게 누적." 상한(영상당 N)만으로는 학생 수가 곱해지는 것을 못 막으므로 **클러스터 캐시가 필수**(아래).

### 5.3 비용 통제 장치 (채택)

1. **클러스터 답변 캐시** — 렌더 전 질문 임베딩 유사도(0.9↑) 비교 후 기존 클립 재생(렌더 0). RAG가 이미 질문 임베딩(`embedding.py`). Pro "유사 질문 클러스터링"(`01-pricing-policy.md:128`)을 아바타 레이어로 확장.
2. **야간 배치 상위 3 클러스터만 렌더** — 영상당 3렌더로 비용 상한 고정 + 가장 많이 겹친 질문에 집중.
3. **이중 상한 + 예산 캡** — 월 2영상 + 영상당 3렌더 + Phase 0 서킷 브레이커(백스톱).

### 5.4 결정 확정 / 잔여

- [x] **상한 단위** → 월 2영상 + **영상 전체에서 3렌더**(학생당 아님). 야간 배치 상위 클러스터.
- [x] **크레딧 차감** → 없음 (월 정액).
- [x] **캐시 유사도 임계값** → 0.9 시작, 베타에서 오답 재생 빈도로 보정.
- [x] **렌더 트리거** → 실시간 금지, 야간 배치.
- [ ] **답변 길이 상한** → 30초 클램프 권장(변동비 상한 고정). 확정 대기.
- [x] **투명성** → 캐시 답변은 원 질문 함께 표시. 권위 있는 답은 학생 질문에 맞춘 텍스트, 아바타는 전달 부가.

## 6. 립싱크 아바타 제공자 전략 (2026-05-27 조사)

핵심 질문: "불쾌한 골짜기 완화가 목적이라면 꼭 비싼 HeyGen이어야 하나?"

### 6.1 상용 제공자 비교 (720p 표준, $1=₩1,380)

| 제공자 | 분당 비용 | 방식 | 한국어 | API | 비고 |
|---|---|---|:---:|:---:|---|
| **HeyGen 표준** | $1 (₩1,380) | 풀/커스텀 아바타 | ○ | ○ | turnkey 최고품질 |
| HeyGen Avatar IV | $4 (₩5,520) | 프리미엄 | ○ | ○ | 비쌈 — **회피** |
| D-ID | ~$3 (₩4,140) | 사진→토킹헤드 | ○ | ○ | HeyGen보다 비쌈 |
| sync.so (lipsync-2) | $2.4~3 | 립싱크 전용 | ○ | ○ | 베이스 영상 필요 |
| DeepBrain AI | 엔터프라이즈 | 풀 아바타 | ◎(한국) | ○ | 한국어 강점, 비용 이점 X |
| **self-host OSS**<br>(MuseTalk / LatentSync 1.5) | **~$0.01~0.02**<br>(₩14~28) | 립싱크 | ○ | 자체 | **50~100배 저렴**, GPU 시간만 |

### 6.2 결론

1. **HeyGen 표준 $1/분은 상용 turnkey 중 오히려 싼 축.** 상용 교체는 비용 답이 아니다.
2. **진짜 비용 바닥 = self-host 오픈소스 립싱크.** 마진은 GPU 시간(~₩20/분)뿐.
3. **불쾌한 골짜기**: 합성 스톡 아바타보다 **교수자 실제 얼굴 베이스 + 립싱크가 더 자연스럽고 더 싸다.** 입만 움직이는 진짜 얼굴.

### 6.3 단계적 전략 (교수자 실제 얼굴 1회 녹화 → 전 구간 립싱크)

- **베타(지금)**: 교수자 footage로 **HeyGen Instant/Photo Avatar 생성** → 그 `avatar_id`로 인트로·아웃트로·Q&A 전 구간. 기존 `HEYGEN_AVATAR_ID_*`·`pick_avatar_id`(`heygen.py:84`) 흐름 재사용. Avatar IV(프리미엄) 금지, 720p. (HeyGen Instant Avatar는 본인 인증·소량 비용 있음 — 확인)
- **추상화(Phase 0~1)**: `LipSyncProvider` 인터페이스 추출 — `heygen.py`가 이미 격리돼 있어 교체가 *config 한 줄*.
- **정식/스케일(Phase 3)**: **같은 원본 footage**를 self-host(MuseTalk/LatentSync)에 투입. Q&A 답변 → 인트로/아웃트로 순 이전. (self-host 손익분기: 상시 GPU 월 $300 ≈ 영상 300분/월 → 교수 ~30명↑. 그 전엔 서버리스 GPU.)
