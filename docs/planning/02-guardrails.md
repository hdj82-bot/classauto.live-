## 1. 도입 배경

### 1.1 위협 시나리오

학생 측 API 남용 가능성:

**시나리오 A**: 챗봇을 학습 외 용도로 사용
- 다른 과제 작성 도움 요청
- 일반 질문 (코딩, 번역, 글쓰기)
- ChatGPT 대용으로 활용
- 결과: Claude API 비용을 ClassAuto가 부담, 학생은 무료로 범용 AI 사용

**시나리오 B**: 영상 자동재생 봇/매크로
- 영상을 자동 재생 상태로 두고 Q&A를 봇으로 호출
- 출석 인증 우회, 학습 스트릭 자동 적립
- 결과: API 무한 호출, 비용 폭주

### 1.2 비용 구조 분석

| 비용 항목 | 학생 수 비례 | 영향도 |

## 2. 4중 가드레일 구조

| 방어선 | 방식 | 차단 대상 |
|:---:|---|---|
| **1차** | 입력 제약 | 보고서·코드 붙여넣기 |
| **2차** | 의미 필터링 (Claude 관련성 판단) | 학습 외 질문 |
| **3차** | 빈도 한도 | 폭주성 사용 |
| **4차** | 이상 탐지 | 봇·매크로 |

이 4중 구조가 모두 작동하면 학생당 비용은 거의 정확히 예측 가능합니다.

## 3. 1차: 입력 제약

### 3.1 정책

```
질문 입력 시 검증:
- 텍스트 ≤ 500자
- 음성 ≤ 30초
- 첨부파일 차단 (PDF, 이미지, 텍스트 파일)
- 초과 시 "질문은 500자 이내로 작성해주세요" 안내
```

### 3.2 차단 효과
- "이 보고서 좀 봐주세요" 식 1만자 붙여넣기 차단
- 학생이 ChatGPT처럼 활용하는 것을 입력 단계에서 봉쇄
- 정상 학습 질문은 거의 500자 안에서 끝남

### 3.3 구현 위치
- 프론트엔드: 입력창에 글자 수 카운터 (500/500)
- 백엔드: API 엔드포인트에서 서버 사이드 검증 (`/api/v1/qa/ask`)
- 음성: STT 변환 후 텍스트 길이도 동일 검증

## 4. 2차: 의미 필터링 (Claude 관련성 판단)

> **정책 변경 (2026-06-11, 교수자 결정)**: 종전의 **임베딩 유사도 하드 컷오프(0.7→0.4)**
> 는 강의 자료(PPT)에 단어가 그대로 없다는 이유로 **정상 강의 질문까지 반복 거부**했다
> (예: 슬라이드 불릿엔 없고 교수자 발화/일반 개념으로만 다룬 "서술어" 질문). 이를 해소
> 하기 위해 관련성 판정 주체를 **임베딩 임계값 → Claude 의미 판단**으로 바꾼다. 가드레일
> 의 목적(학습 외 질문 차단)은 **그대로 유지**되며, "강의 관련 질문만 허용"이 더 정확해
> 지도록 **판정만 똑똑해진 것**이다(완화가 아니라 오거부 해소).

### 4.1 처리 파이프라인

```
질문 처리 파이프라인:
1. 질문 → OpenAI 임베딩으로 강의 자료(슬라이드 PPT + 강의 스크립트) 검색
   - 슬라이드 임베딩(slide_embeddings) + 스크립트 세그먼트(on-the-fly 임베딩)
   - 상위 관련 청크를 RAG 컨텍스트로 구성 (검색·근거 제공용. 임계값 컷오프는 없음)
2. 강의 자료가 전혀 없으면 → Claude 호출 없이 거부 (판단 근거 없음, 비용 0)
3. Claude 호출 — 강의 자료를 1차 근거로 주고 관련성 판단 + 답변을 동시에:
   - 질문이 이 강의가 다루는 주제·개념과 관련 → 전문가 수준으로 매우 자세히 답변
     (자료에 그대로 없어도 강의 주제 범위 안에서 전문 지식으로 보충)
   - 이 강의 주제와 무관(일반 상식·시사·잡담, 다른 주제, 범용 AI 대행 요청)
     → Claude 가 [[OUT_OF_SCOPE]] 만 출력 → 거부 메시지로 치환
```

### 4.2 관련성 판정 기준 (Claude 프롬프트)
- **허용**: 이 강의가 다루는 주제·개념과 관련된 질문. 자료에 직접 없어도, 그 개념을
  더 깊이·더 넓게 묻는 질문 포함.
- **거부**: 일반 상식·시사·잡담 / 강의와 같은 큰 분야라도 이 강의가 다루지 않는 별개
  주제 / 글쓰기·번역·코드 작성 등 범용 AI 요청.
- 임베딩은 더 이상 통과/거부를 결정하지 않는다. 검색·컨텍스트 구성에만 쓴다.

### 4.3 거부 시 학생 화면 메시지

```
🤖 이 질문은 강의 범위 밖이에요.
   강의 내용과 관련된 질문을 부탁드립니다.

   다른 EdTech AI 챗봇은 이런 질문에도 답하지만,
   ClassAuto는 학생이 학습에 집중하도록 설계되었습니다.
```

### 4.4 비용 영향 (트레이드오프)
- 종전: 임베딩 컷오프로 거부 시 Claude 호출 0 → 비용 0.
- 현재: 강의 자료가 있으면 **거부 케이스에도 Claude 1회 호출**(관련성 판단을 Claude 가
  하므로). 강의 자료 자체가 없는 경우만 비용 0 거부.
- 1·3·4차(입력 제약·빈도 한도·이상 탐지)가 폭주성/남용 비용은 계속 방어한다.

## 5. 3차: 빈도 한도

### 5.1 한도 매트릭스

| 항목 | Free | Basic | Pro |
|---|:---:|:---:|:---:|
| 영상당 채팅 Q&A | 20건 | 100건 | 무제한 |
| 학생당 일일 Q&A | — | 30건 | 100건 |
| 학생당 월 Q&A 총량 | — | 500건 | 2,000건 |
| 학생 입력 글자 한도 | 500자 | 500자 | 500자 |
| 영상당 24h 동시 시청 | 30명 | 80명 | 무제한 |
| 학생당 동시 재생 | 1개 | 1개 | 1개 |

### 5.2 카운터 구현 (Redis 권장)

```
실시간 카운터 (Redis 키 패턴):
- counter:user:{학생ID}:daily      → 일 한도 (자정 리셋)
- counter:user:{학생ID}:monthly    → 월 한도 (월초 리셋)
- counter:video:{영상ID}:perEpisode → 편당 한도 (영상 비활성 시 리셋)
- counter:video:{영상ID}:24h        → 24시간 시청자 수 (롤링 24시간)
- counter:user:{학생ID}:concurrent  → 동시 재생 (세션 종료 시 감소)
```

### 5.3 한도 초과 메시지

**일 한도 초과**:
```
오늘 질문 한도(30건)를 모두 사용했어요.
내일 다시 만나요.

내일 자정에 한도가 초기화됩니다.
```

**영상 동시 시청 초과**:
```
이 영상은 현재 80명이 시청 중이에요.
잠시 후 다시 시도해주세요.

대기 시간: 약 ○○분
```

**학생당 동시 재생 초과**:
```
다른 기기에서 이미 영상을 시청 중이에요.
한 학생당 한 화면만 동시 재생 가능합니다.

[다른 기기 종료하고 여기서 시청]
```

### 5.4 정상 학습 사용량 가정
- 학생 1명이 영상 1편당 평균 5~10건 질문
- 일 30건 = 영상 3~6편 학습 가능 (충분)
- 월 500건 = 학기 중 평균 학생 사용 패턴 충분 커버

## 6. 4차: 이상 탐지

### 6.1 자동 차단 조건

```
주기적 패턴 분석 (시간당 1회 배치):
- 학생 1명이 1시간 내 Q&A 20건 이상 → 일시 차단 + 교수자 알림
- 동일 IP 다중 계정 접속 → 의심 플래그
- 24시간 내 영상 5회 이상 재생 시작 → 매크로 의심
- 시청 시간 < 영상 길이의 30% + 출석 통과 시도 → 부정행위 알림
```

### 6.2 데이터 수집
- 이미 기획서의 "딴짓 포착 리포트"·"부정행위 방지" 기능으로 수집되는 데이터 활용
- 임계값만 정의하면 즉시 작동 가능

### 6.3 처리 흐름
1. 백그라운드 Celery 작업이 시간당 실행
2. 의심 패턴 감지 시:
   - 학생: 일시적으로 Q&A 차단 (24시간)
   - 교수자 대시보드 "주의 필요 학습자" 위젯에 알림
   - 학과·학교 관리자에게 보고서 (기관 라이선스)
3. 명백한 매크로 (예: 0.5초 간격 호출) → 즉시 IP 차단

### 6.4 학생 보호 장치
- 자동 차단은 **24시간**만 지속 (영구 차단 아님)
- 학생이 항의 시 교수자 검토 후 즉시 해제 가능
- 오탐 방지를 위해 임계값은 보수적으로 설정

## 7. 데모 페이지 한정 가드레일

데모 페이지(`/demo`)는 회원가입 없이 작동하므로 **별도의 단순화된 가드레일**을 적용합니다.

### 7.1 데모용 한도

| 가드레일 | 실서비스 | 데모 |
|---|---|---|
| 입력 글자 수 | 500자 | 200자 |
| 관련성 판단 | Claude(강의 관련만) | Claude(체험용으로 더 관대) |
| 일일 한도 | 30건 | **세션당 3건** |
| 동시 시청 | 80명 | 무제한 (홍보 우선) |

### 7.2 봇·남용 방지

```
1. Cloudflare Turnstile (visible challenge)
   - 페이지 진입 시 1회 (조용한 검증)

2. Rate limit
   - IP당 데모 세션 1일 3회 제한
   - 같은 질문 반복 차단

3. 비용 캡
   - 서버 사이드 일일 데모 비용 한도 ($100/일 등)
   - 한도 도달 시 "지금 데모가 매우 인기예요. 잠시 후 시도해주세요"
```

### 7.3 데이터 보호

```
- 데모 세션 데이터는 24시간 후 자동 삭제
- 입력 텍스트는 익명화 후 모델 개선용으로만 사용
  (사전 공지 + 체험 시작 버튼 클릭 = 동의)
- IP·fingerprint는 봇 차단 목적 외 사용 금지
```

## 9. 백엔드 구현 체크리스트

새 API 엔드포인트 추가 시 다음 가드레일을 모두 통과해야 합니다:

- [ ] **1차**: 입력 길이 검증 (Pydantic validator)
- [ ] **2차**: 관련성 판단 — 강의 자료(PPT+스크립트)를 근거로 Claude 가 강의 관련만 답변,
  무관 질문은 `[[OUT_OF_SCOPE]]` 로 거부 (`qa.answer_question`)
- [ ] **3차**: Redis 카운터 증감 (트랜잭션)
- [ ] **4차**: 이상 패턴 로그 기록 (Sentry/Prometheus)

가드레일을 우회하는 어떤 코드도 PR에서 머지하지 않습니다.
