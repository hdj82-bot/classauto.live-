## 1. 디자인 철학

### 1.1 화면별 톤 분리 (2026-05-06 갱신)

이전 정책: "학생 영역 = 다크 모드 강제" → **잘못된 일괄 적용**

수정 정책: **영상 유무 기준으로 톤 분리**

| 화면 | 톤 | 이유 |
|---|---|---|
| `/v/[강의ID]` 진입 | **라이트** (`#FAFAF7`) + 골드 | 첫인상 친근감, 신뢰 |
| 학생 회원가입 | **라이트** + 골드 | 폼 가독성, 개인정보 입력 부담 완화 |
| 학생 첫 사용 온보딩 | **라이트 → 다크 전환** | 영상 시청 적응 (마지막 슬라이드만 다크) |
| **영상 시청 화면** | **다크** (`#0A0A0A`) | 영상 몰입 |
| Q&A 사이드 패널 | 다크 (영상 화면의 일부) | 영상 화면 연속성 |
| 집중 경고 · 인터스티셜 퀴즈 | 다크 (오버레이) | 영상 화면 위 |
| `/profile` 학습자 마이페이지 | **라이트** + 골드 | 정보 정리, 가독성 |
| 인증서 PDF | 라이트 (인쇄용) | 출력 가독성 |

### 1.2 핵심 원칙

> **영상이 화면에 있으면 다크, 영상이 없으면 라이트.**

- 영상 몰입 = 다크 (학습 집중)
- 비영상 인터페이스 = 라이트 (친근감·가독성·신뢰)
- 학생도 사람이다 — 모든 화면이 시꺼먼 색이면 사용 부담 증가
- 회원가입 전 첫 화면이 다크면 "이 서비스 안전한가?" 의심 유발

### 1.3 영상 시청 화면 디자인 원칙 (다크 영역)

> **화면의 95% = 영상.** 메뉴는 학습자에게 방해입니다.

- 다크 모드 (`#0A0A0A`) — 영상 몰입
- 컨트롤 UI는 마우스 정지 3초 후 자동 페이드아웃
- 폰트 Pretendard, 크기 살짝 크게 (16px → 17px)
- 모션은 부드럽게 (300ms ease-out)
- YouTube보다는 **Loom / Mux Player** 스타일 참조

### 1.4 비영상 화면 디자인 원칙 (라이트 영역)

- 라이트 모드 (`#FAFAF7` 베이스, `#FFFFFF` 카드)
- 골드 포인트 (`#FFB627`) — 중요 액션·강조
- 그라데이션 메쉬 배경 (선택적, 미세하게)
- 폰트 Pretendard 본문, Paperlogy 헤딩 (히어로 한정)
- 모션은 가벼움 (200-300ms ease-out)

## 2. 사용자 흐름 (전체)

```
[1] /v/[강의ID] 진입 페이지     — 라이트 톤 (첫인상)
[2] 학생 가입 흐름 (3단계)      — 라이트 톤 (폼 입력)
[3] 학생 첫 사용 온보딩 (4슬라이드) — 라이트 → 다크 전환
[4] 영상 시청 화면              — 다크 톤 (몰입)
[5] Q&A 사이드 패널             — 다크 (영상 화면의 일부)
[6] 집중 경고 + 인터스티셜 퀴즈  — 다크 (오버레이)
[7] /profile 학습자 마이페이지  — 라이트 톤 (정보 정리)
```

라이트 → 라이트 → **전환** → 다크 → 다크 → 다크 → 라이트 흐름.

### 2.1 프로토타입 통합 패턴 (2026-05-06 추가)

화면 1~4는 **단일 HTML에 통합**되어 있으며, 화면 간 전환은 부드러운 fade/slide 애니메이션으로 처리합니다. 프로토타입에는 **Demo Navigation** 컨트롤(우측 하단)이 포함되어 4개 화면을 클릭으로 토글할 수 있습니다. 실제 Claude Code 변환 시 이 Demo Navigation은 제거됩니다.

## 3. /v/[강의ID] — 학생 진입 페이지 (회원가입 전) ⭐

### 3.1 톤 — 라이트

**왜 라이트인가**:
- 학생이 카톡/QR로 처음 받은 링크를 클릭하는 순간
- 처음 보는 서비스라 경계심·의심 가능
- 학교 이메일을 입력해야 하는 부담
- 시꺼먼 화면이면 "이거 안전한가?" 추가 의심

### 3.2 화면 구성

```
[배경: 라이트 #FAFAF7, 미세한 그라데이션 메쉬 선택적]

ClassAuto                                       [언어 KO]

┌────────────────────────────────────┐
│ (하) 하두진 교수님이 보낸 강의입니다  │   [✓인증]
│      경기대학교 · 중국어문법의 이해   │
└────────────────────────────────────┘

┌────────────────────────────────────┐
│ 📖                                  │
│      3주차 · LESSON 07              │
│      把자문(把字句) 입문             │
│      경기대학교 · 중국어문법의 이해   │
│                                    │
│   ⏱ 5분 12초   👥 23명 학습 중   📈 82% │
└────────────────────────────────────┘

┌────────────────────────────────────┐
│ 시청 조건                            │
│  ✓ 학교 이메일 인증 (.ac.kr)         │
│ 또는                                 │
│    학습 코드 입력  [ABCD-1234]      │
└────────────────────────────────────┘

[ 📧 학교 이메일로 시작 →]   ← 골드 채움
[ 🔒 학습 코드로 시작 →]     ← outline

처음 사용하시나요? [1분 안내 보기 →]

────────────────────────────────────
ClassAuto는 학생 학습 도구입니다.
광고를 사용하지 않으며, 시청 데이터는 해당 강의의
교수자만 볼 수 있습니다.
학생 데이터 보호 정책 보기 →
```

### 3.3 핵심 요소 (프로토타입 검증 완료)
- 교수자 신원으로 신뢰 — "하두진 교수님이 보낸 강의입니다"
- 강의 메타 카드 (3주차, LESSON 07, 把자문, 5분 12초, 23명 학습 중, 82%)
- 진입 방식 두 가지 (이메일 / 학습 코드) 동등 노출
- 처음 사용자를 위한 "1분 안내" 링크
- 데이터 보호 안내 (하단)

### 3.4 인증 옵션 분기
- **학교 이메일**: `.ac.kr`, `.edu` 등 도메인 화이트리스트 매칭
- **학습 코드**: 교수자가 발급한 4-4 코드 (예: ABCD-1234). 임시 익명 접속

### 3.5 라이트 톤 디자인 디테일
- 배경: `#FAFAF7` (메인 사이트와 동일)
- 카드: `#FFFFFF` (그림자 살짝)
- 텍스트: `#0A0A0A` 본문, 보조는 muted
- 골드: 라이트 배경에서 `#B88308` 어둡게 사용

## 4. 학생 회원가입 흐름 (3단계)

### 4.1 톤 — 라이트

폼 입력 가독성. 진입 페이지와 일관성.

### 4.2 3단계 구조 (프로토타입 검증 완료)

**Step 1/3: 학교 이메일 입력**
- 진행 도트 `● ○ ○`
- Paperlogy 헤딩: "학교 이메일을 입력해주세요"
- 서브: "ClassAuto는 학교 이메일로만 가입할 수 있어요"
- 입력란: `name@kgu.ac.kr` placeholder
- 도메인 검증 표시 (`.ac.kr` 매칭)
- CTA: "인증 메일 보내기" (골드 채움)
- 보조: "또는 [학습 코드로 시작]"

**Step 2/3: 인증 메일 확인 안내**
- 진행 도트 `● ● ○`
- 일러스트: 편지 아이콘 그라데이션 SVG (electric, 120px)
  - 봉투 위로 골드 점 떠오르는 미세 애니메이션
- Paperlogy 헤딩: "메일을 보내드렸어요"
- 본문: "name@kgu.ac.kr로 인증 링크를 보냈어요"
- 보조 카드: "💡 스팸 메일함도 확인해주세요"
- "다시 보내기" 링크 (60초 카운트다운)

**Step 3/3: 추가 정보 입력**
- 진행 도트 `● ● ●`
- Paperlogy 헤딩: "거의 다 됐어요"
- 폼 필드 5개:
  1. 이름 (필수)
  2. 학번 (필수)
  3. 학교 (자동 감지·잠금) — 이메일 도메인 기반
  4. 학과 (자동완성 콤보)
  5. 학년 (라디오: 1/2/3/4/대학원)
- **데이터 정책 사전 고지 3카드** (ClassAuto 차별점):
  - 📊 수집 데이터: 영상 시청·Q&A·출석
  - 👁️ 열람 권한: 해당 강의 교수자만
  - 🗑️ 삭제 정책: 졸업 후 자동
- 동의 체크박스: "데이터 처리 방침에 동의합니다 (필수)"
- "가입 완료" 버튼 (체크박스 활성화 시에만)

### 4.3 단계 전환
- 단계 사이 슬라이드 전환 (좌→우 360ms ease-out)
- 진행 도트 채워지는 애니메이션 (200ms)
- Step 2, 3에 좌상단 "← 이전 단계로" 링크

### 4.4 레이아웃 일관성 (2026-05-06 추가)
- 모든 단계 동일한 `max-width` (480-560px)
- 데이터 정책 3카드는 카드 폭 안에 들어와야 함 (overflow 주의)
- 학년 라디오 5칸이 카드 폭 초과하면 칩 크기 축소 또는 2x3 그리드

## 5. 학생 첫 사용 온보딩 (4슬라이드)

### 5.1 톤 — 라이트 → 다크 전환

- 슬라이드 1~3: **라이트** (회원가입 흐름 연속)
- 슬라이드 3 → 4 사이: **1초 페이드 전환** (라이트 → 다크)
- 슬라이드 4: **다크** (영상 시청 적응)

### 5.2 4슬라이드 구성

**슬라이드 1 (라이트, 8초)**
- 일러스트: 영상 재생 아이콘 그라데이션 SVG (electric)
- 헤딩 (Paperlogy): "영상을 보면서 배워요"
- 본문: "원하는 시간에 멈추고 다시 볼 수 있어요"

**슬라이드 2 (라이트, 8초)**
- 일러스트: 채팅 말풍선 그라데이션 SVG (violet)
- 헤딩: "궁금한 건 물어보세요"
- 본문: "AI 도우미가 강의 자료 기반으로 답해줍니다"

**슬라이드 3 (라이트, 8초)**
- 일러스트: 올빼미 마스코트 SVG (회갈색, "걱정" 표정)

**슬라이드 4 (다크, 6초)**

## 8. 집중 경고 3단계 + 인터스티셜 퀴즈 — 다크 (오버레이)

영상 화면 위에 등장하는 모달이라 모두 다크 톤.

### 8.1 1단계 경고 (5초 무반응)
- 화면 가장자리 부드러운 펄스 (회갈색 글로우)
- 좌하단 토스트: "🦉 잠깐, 잘 따라오고 계신가요?"
- 영상은 계속 재생 (방해 최소화)
- 클릭/키보드/마우스 시 즉시 해제

### 8.2 2단계 경고 (15초 무반응)
- 영상 반투명 오버레이로 흐려짐
- 큰 올빼미 (200px, 놀란 표정)
- "오늘은 좀 피곤하신가요? 잠시 쉬어가도 좋아요"
- "계속 볼게요" 클릭 필수

### 8.3 3단계 경고 (30초 또는 누적 3회)
- 영상 완전 정지 + 다크 오버레이
- 올빼미 (또렷한 응시)
- 3가지 액션: [처음부터 다시 보기] / [짧은 퀴즈로 점검] / [5분 후 다시 시도]
- 누적 기록 → 교수자 "주의 필요 학습자" 자동 등록

### 8.4 인터스티셜 퀴즈 (영상 중간 자동)

把자문 시연 기준:
```
🦉 잠깐 확인! (10초)
방금 들으신 내용 중,
把자문에서 把 뒤에 오는 명사는 어떤 의미인가요?

○ A. 동작의 주체
● B. 동작의 대상 (처치 대상)  ← 정답
○ C. 시간 표현
○ D. 장소 표현

[정답 확인]
```

- 정답 시: ✓ "잘 이해하고 계세요!" → 2초 후 영상 재개
- 오답 시: "정답은 B예요. 슬라이드 X-X를 다시 볼까요?" + 점프 옵션
- 오답 기록 → 교수자 "취약점 히트맵" 누적
