# 보조 페이지 기획

> **상태**: 확정 · 2026-05-05
> **연관 문서**: [03-sitemap.md](./03-sitemap.md)

랜딩·데모·교수자·학생 외 모든 보조 페이지의 기획. 각 페이지가 ClassAuto의 어떤 가치를 어떤 페르소나에게 전달하는지 명시.

---

## 1. /use-cases (활용 사례) 🔴 1순위

### 1.1 페이지 목적
교수자가 "내 분야에서도 쓸 수 있나?" 의문 해소.

### 1.2 구성
- 학과별 시나리오 6~8개 카드 (어학·인문·이공계·사회과학·예체능·실험실습)
- 각 카드:
  - 가상 교수님 페르소나 프로필
  - **Before / After** 비교
  - 핵심 기능 3개 강조
- 어흥 교수님의 **실제 중국어과 사례를 앵커 케이스**로 (가장 상단 큰 카드)
- "다른 분야 사례 추가 예정" 안내

### 1.3 카드 예시 구조

```
[분야 아이콘] 사회과학·국제학
김○○ 교수님 (가상)
A 대학 정치외교학과

Before:
"매주 시사 강의를 영상으로 만들고 싶었지만,
 매번 새 자료를 만들 시간이 없었습니다."

After:
"PPT 한 장 + AI 스크립트 = 12분 만에 완성.
 학생들의 출석률이 78% → 92%로 올랐습니다."

핵심 기능:
✓ 이중 TTS (한국어 + 영어)
✓ 인터스티셜 퀴즈
✓ 취약점 히트맵
```

### 1.4 왜 1순위
베타 단계에서 학계 보급 시 동료 학자가 가장 먼저 묻는 질문이 "내 분야에 맞나?". 이 페이지가 없으면 영업이 1:1로 설명해야 함.

## 2. /trust (학생 데이터 보호) 🔴 1순위

### 2.1 페이지 목적
학생 데이터 정책을 **이해하기 쉬운 시각적 페이지**로 노출. 법적 문서(/legal/privacy)와는 별개.

### 2.2 핵심 메시지

```
ClassAuto는 학생을 위한 도구입니다.

📊 어떤 데이터를 수집하나요?
- 영상 시청 기록 (몇 분 봤는지)
- Q&A 질문 내용
- 퀴즈 응답
- 출석 정보

👁️ 누가 볼 수 있나요?
- 해당 강의 교수자만
- 광고에 사용하지 않음
- 제3자에게 판매하지 않음

🗑️ 언제 삭제되나요?
- 졸업 후 자동 삭제
- 학생이 요청 시 즉시 삭제

🌍 어디에 저장되나요?
- 한국 데이터센터
- KISA 가이드라인 준수
```

### 2.3 학생 권리
- 자신의 데이터 다운로드 요청
- 삭제 요청 (즉시 처리)
- 사용 내역 조회
- 문의: privacy@classauto.live

### 2.4 디자인
- 메인 사이트 톤
- 큰 일러스트 (그라데이션 SVG)
- 각 항목에 자세히 보기 모달

## 3. /security (보안 정책) 🔴 1순위

### 3.1 페이지 목적
**학과장·전산팀이 검토할 수 있는 기술 자료**. 기관 라이선스 도입 검토 시 필수.

### 3.2 콘텐츠 구조

```
1. 데이터 암호화
   - 전송: TLS 1.3
   - 저장: AES-256
   - DB: 컬럼 단위 암호화 (개인정보)

2. 접근 통제
   - RBAC (Role-Based Access Control)
   - 교수자 → 자기 강의만
   - 학과장 (Phase 4) → 학과 단위 조회만

3. 인시던트 대응
   - 24시간 모니터링 (Sentry + Prometheus)
   - 침해 사고 시 72시간 내 통지 (개인정보보호법)
   - 분기별 보안 감사

4. 외부 감사
   - (계획) ISMS-P 인증
   - (계획) SOC 2 Type II

5. 한국 법률 준수
   - 개인정보보호법
   - 정보통신망법
   - KISA 가이드라인

6. API 보안
   - OAuth 2.0
   - Rate limiting
   - API 키 회전 정책
```

### 3.3 다운로드 자료 (PDF)
- 보안 백서
- 데이터 처리 약관
- DPA (Data Processing Agreement, 기관용)

### 3.4 문의
- security@classauto.live
- 보안 취약점 제보: 책임 공개 정책 명시

## 4. /beta-apply (베타 신청 폼) 🔴 1순위

### 4.1 폼 항목

```
1. 이름 *
2. 학교 *
3. 학과 *
4. 직책 (교수/조교수/강사 등) *
5. 학교 이메일 * (.ac.kr 도메인 검증)
6. 가르치는 과목 (자유 입력) *
7. 학생 수 (선택)
8. 시작 희망 시기 *
   - 즉시
   - 다음 학기
   - 미정
9. 어떻게 알게 되셨나요?
   - 동료 추천
   - 학회/세미나
   - 검색
   - 기타
10. 추가로 하고 싶은 말씀 (선택)
```

### 4.2 제출 후 흐름

```
✓ 신청이 접수되었습니다

3 단계 안내:
1. 신청 검토 (1-2 영업일)
2. 학교 이메일 인증 메일 발송
3. 교수자 페르소나 등록 안내

문의: beta@classauto.live
```

자동 이메일:
- 즉시: 신청 확인 메일
- 1-2일 후: 승인/추가 정보 요청 메일

### 4.3 검증 정책
- 학교 이메일 도메인 화이트리스트
- 미매칭 시 명함·재직증명 첨부 옵션
- 의심 신청 (비대학 도메인, 중복 IP) → 수동 검토

### 5.1 폼 항목

```

6. 도입 검토 단계 *
   - 정보 수집 중
   - 내부 논의 중
   - 의사결정자 결재 단계
   - 즉시 도입 가능
7. 예상 교수자 수 *
8. 예상 학생 수 *
9. 검토 중인 LMS *
   - Moodle / Canvas / Blackboard / KLAS / eClass / 기타
10. 통화 가능 시간 *
11. 추가 문의 사항 (자유 입력)
```

### 5.2 제출 후 흐름

```
✓ 견적 문의가 접수되었습니다

다음 단계:
1. 영업 담당자 확인 (1 영업일)
2. 화상 미팅 일정 조율
3. 맞춤 제안서 발송
4. 파일럿 도입 또는 정식 계약

긴급 문의: contact@classauto.live
직통: 02-XXXX-XXXX (예정)
```

### 5.3 자동화
- HubSpot 또는 Salesforce CRM 연동
- 영업 담당자 자동 배정 (지역별)
- Calendly 캘린더 링크 자동 발송

## 6. /legal/terms (이용약관) 🟡 2순위

### 6.1 결제 시작 전 필수 게재

표준 SaaS 이용약관 + 교육 도구 특수 조항:
- 서비스 정의 (ClassAuto = 교육용 SaaS)
- 사용자 분류 (교수자 / 학생 / 기관)
- 콘텐츠 소유권 (교수자 콘텐츠는 교수자 소유)
- AI 생성물 (학교 저작권 정책 반영 안내)
- 결제·환불 (가격 정책과 동일)
- 해지 (60일 데이터 보존)
- 분쟁 해결 (한국 법률 적용, 서울중앙지방법원)
- 약관 변경 통지 (최소 30일 전)

### 6.2 작성 방법
- 법무 검토 받기 (변호사 또는 LegalZoom 한국)
- 표준 SaaS 약관 + 교육 도구 조항 추가
- 학생용·교수자용·기관용 분리 가능

## 7. /legal/privacy (개인정보처리방침) 🟡 2순위

### 7.1 한국 개인정보보호법 준수

필수 항목:
- 개인정보 처리 목적
- 수집 항목 (학생: 이름·학교 이메일·학번 / 교수자: + 소속)
- 보유 기간
- 제3자 제공 (없음 명시)
- 위탁 처리 (HeyGen, ElevenLabs, Anthropic, OpenAI 등)
- 개인정보 처리 위탁 (해외 이전 포함)
- 정보주체 권리
- 쿠키 사용
- 개인정보 보호책임자 연락처

### 7.2 학생 데이터 보충 약관
- 미성년자 (만 14세 미만) 동의 절차
- 교육 목적 사용 한정
- 광고·마케팅 미사용 명시
- 졸업 후 자동 삭제
